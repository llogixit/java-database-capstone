As a Patient, I want to log into the portal with my email and password so that I can securely manage my doctor appointments and bookings.

Acceptance Criteria
Login Form  
The portal must provide a login form where patients can enter their registered email and password.

Authentication Validation  
The system must validate credentials against stored patient records before granting access.

Access to Bookings  
Once logged in, patients must be able to view, edit, cancel, or reschedule their existing bookings.

Secure Session Handling  
A secure session must be established upon login, with automatic timeout after inactivity.

Error Feedback  
If login fails, the system must display a generic error message without revealing whether the email or password was incorrect.

Audit Logging  
All patient login attempts (successful or failed) must be logged for monitoring and analytics.

Priority
High — Core functionality for patient access and booking management.

Story Points
5–7 points (depending on complexity of session management and booking integration).
