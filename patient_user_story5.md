As a Patient, I want to log into the portal and book an hour‑long appointment with a doctor so that I can consult securely and manage my healthcare needs.

Acceptance Criteria
Login Access  
Patients must log in with valid credentials before booking appointments.

Doctor Selection  
Patients must be able to browse and select a doctor from the available list.

Appointment Duration  
The booking system must allow patients to select a one‑hour appointment slot.

Availability Check  
The system must verify the doctor’s availability before confirming the booking.

Confirmation Message  
After booking, the system must display a confirmation message and send an email notification to the patient.

Booking Management  
Patients must be able to view, reschedule, or cancel the appointment after booking.

Audit Logging  
All bookings must be logged with patient ID, doctor ID, and timestamp for compliance.

Priority
High — Core functionality for patient‑doctor interaction.

Story Points
6–8 points (depending on complexity of scheduling logic and notifications).
