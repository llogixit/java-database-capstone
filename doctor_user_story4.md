As a Doctor, I want to mark my unavailability in the portal so that patients can only book appointments during my available slots.

Acceptance Criteria
Unavailability Settings  
The portal must allow doctors to mark specific dates and times as unavailable.

Calendar Integration  
Marked unavailability must automatically update the doctor’s appointment calendar.

Patient View Restriction  
Patients must only see available slots when booking; unavailable times must be hidden or marked as blocked.

Real-Time Updates  
Any changes to unavailability must reflect immediately in the patient booking interface.

Error Handling  
If unavailability overlaps with existing appointments, the system must prompt the doctor to reschedule or cancel those appointments first.

Audit Logging  
All unavailability updates must be logged with doctor ID, timestamp, and affected slots for compliance.

Priority
High — Essential for accurate scheduling and patient experience.

Story Points
5–7 points (depending on complexity of calendar sync and conflict resolution).
