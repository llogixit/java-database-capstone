As an Admin, I want to add doctors to the portal so that they can access and manage patient-related information securely within the system.

Acceptance Criteria
Doctor Registration Form  
The portal must provide a form where the admin can input doctor details (name, specialization, contact info, credentials).

Role Assignment  
Each doctor added must automatically be assigned the "Doctor" role with appropriate permissions.

Credential Verification  
The system must validate required fields (e.g., medical license number) before allowing the doctor to be added.

Confirmation Message  
After successful addition, the system must display a confirmation message and log the event.

Error Handling  
If required fields are missing or invalid, the system must prevent submission and show clear error messages.

Audit Logging  
All doctor additions must be recorded in the audit log for compliance and tracking.

Priority
High — Core functionality for system usability.

Story Points
5–7 points (depending on complexity of credential validation and role assignment).
