As a Patient, I want to view a list of doctors without logging in so that I can explore available options before deciding to register on the portal.

Acceptance Criteria
Public Doctor Directory  
The portal must display a list of doctors (name, specialization, and basic profile) accessible without login.

Limited Information  
Only non-sensitive details (e.g., name, specialization, years of experience) should be visible; contact details and schedules must remain hidden until registration.

Search and Filter  
Patients must be able to search and filter doctors by specialization, location, or availability.

Registration Prompt  
When patients attempt to access restricted details (e.g., appointment booking, contact info), the system must prompt them to register or log in.

Responsive Design  
The doctor list must be accessible across devices (desktop, tablet, mobile).

Audit Logging  
Public views should be logged for analytics, but without storing personal patient identifiers.

Priority
Medium — Important for patient onboarding but secondary to core security features.

Story Points
5–6 points (depending on complexity of search/filter and responsive design).
