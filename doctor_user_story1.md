As a Doctor, I want to log into the portal with my credentials so that I can securely manage my appointments and patient consultations.

Acceptance Criteria
Login Form  
The portal must provide a login form where doctors can enter their registered username/email and password.

Authentication Validation  
The system must validate credentials against stored doctor records before granting access.

Appointment Dashboard  
Once logged in, doctors must be able to view, accept, reschedule, or cancel patient appointments.

Secure Session Handling  
A secure session must be established upon login, with automatic timeout after inactivity.

Error Feedback  
If login fails, the system must display a generic error message without revealing whether the username or password was incorrect.

Audit Logging  
All doctor login attempts (successful or failed) must be logged for monitoring and compliance.

Priority
High — Core functionality for doctor access and appointment management.

Story Points
5–7 points (depending on complexity of session management and appointment integration).
